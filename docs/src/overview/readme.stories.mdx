We love [react-hooks](https://code.fb.com/open-source/react-hooks) and willing to share our [react-native](https://facebook.github.io/react-native) ui components built in [function components](https://reactjs.org/docs/components-and-props.html#function-and-class-components) using hooks. Also the `dooboo-ui` is built on top of our favorite stacks like [emotion](https://emotion.sh/docs/@emotion/native), [typescript](https://www.typescriptlang.org/), [jest](https://jestjs.io), [react-testing-library](https://testing-library.com/docs/react-testing-library/intro), [expo](https://expo.io), [storybook](https://storybook.js.org), and so on.

## Philosophy

`dooboo-ui` aims to provide easy, lightweight, and flexible UI components with the full control of `theme` variants and responsive layout.

## Install

### Expo

```sh
expo install dooboo-ui @emotion/react @emotion/native react-native-vector-icons react-native-gesture-handler react-native-svg expo-screen-orientation @expo/match-media
```

### React Native CLI

```sh
# If using yarn
yarn add dooboo-ui @emotion/react @emotion/native react-native-vector-icons react-native-gesture-handler react-native-svg expo-screen-orientation @expo/match-media

# Install expo modules
npx install-expo-modules@latest
```

### dooboo-cli

You can easily set boilerplate code with `dooboo-ui` by using [dooboo-cli](https://github.com/dooboolab/dooboo-cli).
Please read the `doobo-cli` documents.

## Usage

We aim to support `react-native` ui components in all platforms and we are currently targeting `iOS`, `android` and `web`. If you read [issue on plan for unifying dooboo-ui-legacy](https://github.com/dooboolab-community/dooboo-ui-legacy/issues/194), you can see in more detail how we want to drive this project.

From version `0.2.0`, we recommend you to wrap your App with `DoobooProvider` in order to apply theme and uis provided by `dooboo-ui`.
The original `ThemeProvider` is embedded in `DoobooProvder`.

```tsx
import {DoobooProvider} from 'dooboo-ui';

<DoobooProvider>
  <App />
</DoobooProvider>;
```

## Theming

See [how to theme your app](?path=/docs/overview-theming--page).
The major changes in `0.2.0` is that we recommend using `DoobooProvider` instead of `ThemeProvider` in your application.

## Install Font (Recommended)

In `dooboo-ui`, the default font is [pretendard](https://github.com/orioncactus/pretendard). We use `Pretandard`, `Pretendard-Bold`, `Pretendard-Thin` as font families.
Fonts are installed automatically when installing `dooboo-ui` from `0.2.1`.
However, you should check the `assetLoaded` from `DoobooProvider` to make sure that the fonts are loaded.

```tsx
import {useDooboo} from 'dooboo-ui';

const {assetLoaded} = useDooboo();

if (assetLoaded) {
  /// Loading
  return ...
}

return <Main/>
```

## Using font icons

[Phosphoricons](https://phosphoricons.com) can be used in `dooboo-ui`. You can use `Icon` component to use the icons.

```js
import {Icon} from 'dooboo-ui';

...

<Icon name="..." color="#AAA" size={32} />;
```

## Troubleshoot

### Workaround when you face error in expo web

You need to set webpack for using "dooboo-ui" in expo-web.

#### 1. Install @expo/webpack-config in your expo's project.

```sh
yarn add @expo/webpack-config
```

#### 2. Create webpack.config.js in root path and Add below code.

See [issue](https://forums.expo.io/t/error-when-running-expo-start-web/33096/3) below for more details.

```javascript
const createExpoWebpackConfigAsync = require('@expo/webpack-config');

module.exports = async function (env, argv) {
  const config = await createExpoWebpackConfigAsync(
    {
      ...env,
      babel: {
        dangerouslyAddModulePathsToTranspile: ['dooboo-ui'],
      },
    },
    argv,
  );
  return config;
};
```
